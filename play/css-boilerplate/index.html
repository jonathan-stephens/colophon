Advanced Features

Container Queries

This boilerplate uses container queries for responsive design. The grid layouts automatically adapt based on their container width, not the viewport width.

container-type: inline-size;

CSS Nesting

Modern CSS nesting allows for more maintainable stylesheets with parent-child relationships clearly defined in the code structure.

.button {
    padding: var(--space-md);

    &:hover {
        transform: translateY(-1px);
    }

    &--primary {
        background: var(--color-interactive-primary);
    }
}

View Transitions

The boilerplate includes setup for the View Transitions API, enabling smooth page transitions and morphing animations between states.

view-transition-name: custom-name;

CSS Layers

Uses cascade layers for better specificity management and style organization:

  • foundation.reset - CSS reset
  • foundation.tokens - Design tokens
  • foundation.base - Base styles
  • components.base - Component foundations
  • components.variants - Component variations
  • composition.layouts - Layout patterns
  • transition.effects - Animations
