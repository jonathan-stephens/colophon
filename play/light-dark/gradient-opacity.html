Implementation Guide

Setting Up Opacity Variants

Convert your hex colors to RGB format to enable opacity control:

body {
  /* Extract RGB values from hex colors */
  --brand-rgb-mid: 59 130 246;
  
  /* Create opacity variants */
  --brand-alpha-10: rgb(var(--brand-rgb-mid) / 0.1);
  --brand-alpha-20: rgb(var(--brand-rgb-mid) / 0.2);
  --brand-alpha-30: rgb(var(--brand-rgb-mid) / 0.3);
  /* ... and so on */
}

Defining Semantic Gradients

Create reusable gradient variables that reference your color scales:

body {
  /* Linear gradients */
  --gradient-brand-diagonal: linear-gradient(
    135deg,
    var(--brand-light),
    var(--brand-dark)
  );
  
  /* Radial gradients */
  --gradient-brand-radial: radial-gradient(
    circle at center,
    var(--brand-light),
    var(--brand-dark)
  );
  
  /* Multi-color gradients */
  --gradient-brand-accent: linear-gradient(
    135deg,
    var(--brand-mid),
    var(--accent-mid)
  );
  
  /* Overlay gradients with opacity */
  --gradient-overlay-dark: linear-gradient(
    to bottom,
    rgb(0 0 0 / 0.1),
    rgb(0 0 0 / 0.9)
  );
}

Usage Best Practices

  • Opacity for Layers: Use 10-30% for subtle backgrounds, 50% for overlays, 70-90% for modal backdrops
  • Gradients for Emphasis: Reserve gradients for hero sections, CTAs, and premium features
  • Performance: Limit complex gradients on frequently repainting elements
  • Accessibility: Ensure text over gradients maintains sufficient contrast (WCAG AA minimum 4.5:1)
  • Consistency: Use pre-defined gradient variables instead of ad-hoc gradients

Browser Support & Fallbacks

RGB with alpha channel syntax is widely supported. For maximum compatibility: 

/* Fallback pattern */
.element {
  /* Fallback for older browsers */
  background: var(--brand-light);
  
  /* Modern browsers with alpha support */
  background: var(--brand-alpha-20);
}

/* Gradient fallbacks */
.hero {
  background: var(--brand-mid); /* Solid fallback */
  background: var(--gradient-brand-diagonal); /* Gradient */
}
