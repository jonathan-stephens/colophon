Overview

This demo showcases a scalable color system using CSS's light-dark() function. The system allows you to:

  • Define color themes with a consistent 7-step scale (lightest → darkest)
  • Create semantic color mappings that automatically adapt to light/dark mode
  • Swap entire color themes while maintaining visual hierarchy and accessibility
  • Write CSS once and have it work across all themes and color schemes

Try it: Use the controls above to switch between themes and toggle dark mode. Notice how the interface maintains consistent contrast and hierarchy across all combinations.

How It Works

Step 1: Define Color Scales

Each theme defines seven color variables from lightest to darkest:

[data-theme="ocean"] {
  --lightest: #eff6ff;
  --lighter: #dbeafe;
  --light: #93c5fd;
  --mid: #3b82f6;
  --dark: #1d4ed8;
  --darker: #1e40af;
  --darkest: #172554;
}

Step 2: Create Semantic Mappings

Use light-dark() to create semantic variables that invert automatically:

:root {
  color-scheme: light dark;
  
  /* Backgrounds use light colors in light mode, dark in dark mode */
  --bg-base: light-dark(var(--lightest), var(--darkest));
  --bg-subtle: light-dark(var(--lighter), var(--darker));
  
  /* Text uses dark colors in light mode, light in dark mode */
  --text-primary: light-dark(var(--darkest), var(--lightest));
  --text-secondary: light-dark(var(--darker), var(--lighter));
  
  /* Interactive elements stay consistent */
  --interactive-base: var(--mid);
  --border-default: light-dark(var(--light), var(--dark));
}

Step 3: Use Semantic Variables

Build your components using semantic variables instead of raw colors:

.card {
  background: var(--bg-surface);
  color: var(--text-primary);
  border: 1px solid var(--border-default);
}

button {
  background: var(--interactive-base);
  color: white;
}

Theme Independence

Components don't know or care which theme is active. They only reference semantic variables, making them infinitely reusable across different color palettes.

  • Add new themes without touching component CSS
  • Color logic lives in one place, not scattered across components

Accessibility

Performance

Implementation Guide

1. Set up color-scheme

Enable automatic dark mode detection:

:root {
  color-scheme: light dark;
}

2. Define your themes

Create a 7-step scale for each theme. Use a consistent approach to ensure proper contrast:

  • Lightest: Very subtle background tints
  • Lighter: Secondary backgrounds, hover states
  • Light: Borders, disabled states
  • Mid: Interactive elements, primary actions
  • Dark: Hover states for interactives
  • Darker: Secondary text
  • Darkest: Primary text, high-contrast elements

3. Create semantic mappings

Map your scale to purpose-driven variables using light-dark(). The first argument is for light mode, the second for dark mode.

4. Use semantic variables everywhere

Never reference raw color values in components. Always use semantic variables like var(--text-primary) or var(--bg-surface).

5. Add smooth transitions

Include transitions on color-affected properties for smooth theme changes:

body {
  transition: background-color 0.3s ease, color 0.3s ease;
}

Browser Support

The light-dark() function is supported in:

  • Chrome/Edge 123+
  • Firefox 120+
  • Safari 17.5+

For older browsers, consider using a fallback strategy with CSS custom properties and media queries, or use a PostCSS plugin to transform the syntax.
